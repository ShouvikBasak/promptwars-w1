# General Workspace Rule

This is a general rule that applies to the workspace. The Agent should consistently follow these guidelines throughout all interactions in the repository.

## 1. Code Quality & Formatting
* Write clean, maintainable, and self-documenting code.
* Prioritize readability over excessive cleverness or brevity.
* Keep functions and methods small, focusing on a single responsibility.
* Follow standard idiomatic patterns for the language in use.

## 2. Documentation & Comments
* Include docstrings and comments for all non-trivial logic.
* Comments should explain the *why* rather than the *what*.
* Update relevant documentation (.md files) whenever structural code changes are made.

## 3. Communication
* Keep responses concise and focused directly on solving the user's issue.
* Provide explanations only when necessary or when explicitly requested.
* Use Markdown formatting for code snippets, lists, and highlighting important keywords.
